# 📊 Analizador BigO (Multi-language)

> Una herramienta de análisis de rendimiento diseñada para evaluar la eficiencia temporal y espacial de algoritmos escritos en C++, Python y Java, facilitando la creación de código escalable y optimizado.

---

## 🎯 El Desafío
En el desarrollo de software a gran escala, un algoritmo funcional no siempre es un algoritmo eficiente. 
* **Problema:** La falta de herramientas accesibles para medir el impacto de la complejidad algorítmica en diferentes lenguajes durante la fase de desarrollo.
* **Solución:** Crear una plataforma que procese métricas de rendimiento y ayude a identificar cuellos de botella mediante el análisis de la complejidad computacional.

## 🛠️ Tecnologías Utilizadas
* **Core de Procesamiento:** Java (para la lógica robusta de backend y cálculo de métricas).
* **Visualización y Métricas:** JavaScript (para la generación de reportes dinámicos).
* **Lenguajes Soportados para Análisis:** C++, Python, Java.

## ✨ Características Principales
* ✅ **Análisis Multi-lenguaje:** Evaluación comparativa de eficiencia entre los lenguajes más utilizados en la industria.
* ✅ **Métricas de Escalabilidad:** Determinación de la robustez del código frente al crecimiento de datos de entrada.
* ✅ **Reportes de Rendimiento:** Generación de datos visuales que facilitan la toma de decisiones para la optimización (Refactoring).
* ✅ **Interfaz Intuitiva:** Herramienta diseñada para ser integrada fácilmente en el flujo de trabajo de un desarrollador.

## 🔬 ¿Qué analiza el sistema?
El analizador se enfoca en los pilares del análisis de algoritmos:
1. **Complejidad Temporal:** Tiempo de ejecución en relación al tamaño de la entrada.
2. **Complejidad Espacial:** Uso de memoria y recursos del sistema durante el procesamiento.
3. **Puntos Críticos:** Identificación de bucles anidados o recursividad ineficiente.
